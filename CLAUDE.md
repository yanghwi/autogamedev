# CLAUDE.md — Midnight Bestiary: Autonomous Game Evolution

## 철학

Karpathy autoresearch 정신. 인간은 목표와 최소 원형만 제공한다.
에이전트가 게임을 설계, 구현, 평가, 진화시킨다.

## 인간이 정한 것 (이것만)

1. **장르**: 싱글플레이어 오토배틀러
2. **미학**: Mother/EarthBound 풍 (다크, 사이키델릭, 레트로 픽셀)
3. **목표**: "재미있는 게임"을 만들어라

## 에이전트가 결정하는 것 (나머지 전부)

- 몬스터 수, 종류, 스탯 체계
- 전투 시스템 (턴제? 실시간? 하이브리드?)
- 팀 구성 방식 (드래프트? 수집? 진화?)
- 진행 구조 (웨이브? 스테이지? 무한?)
- 시너지/특성/스킬 시스템의 존재 여부와 형태
- 난이도 곡선
- UI/UX 구조
- 지표의 추가/수정/폐기
- 스프라이트 생성 (기존 287장은 참고 자료)

## 자율 루프

```
1. evaluate.py로 현재 상태 측정
2. 지표 분석 → 개선 가설 생성
3. game.py 수정
4. 재측정 → 개선되었는가?
   → Yes: git commit + 다음 가설
   → No: git checkout -- game.py + 가설 수정
5. 반복
```

## 파일 구조

```
game.py        ← 게임 로직 (에이전트의 주 작업 대상)
evaluate.py    ← 재미 지표 측정 (에이전트가 지표도 진화시킴)
CLAUDE.md      ← 이 파일 (에이전트 자신이 필요하면 수정)
sprites/       ← 참고용 기존 스프라이트 (287장)
results/       ← 실험 결과 아카이브
```

## 원칙

- **빌드 없음**: `python3 evaluate.py` 한 줄로 평가
- **외부 의존성 없음**: Python 표준라이브러리만
- **git이 실험 노트**: 매 유의미한 변경마다 커밋
- **단일 source of truth**: game.py가 게임 전체
- **과감하게 실험**: 실패해도 git checkout으로 복구

## 기존 자산 (참고용)

- `client/` 디렉토리: 기존 React+TS 프로토타입 (참고만)
- `client/public/sprites/`: 287개 EarthBound풍 PNG (스타일 참고)
- 기존 게임 설계 (134몬스터, 8카테고리, 10웨이브 등)는
  하나의 가능한 설계안일 뿐, 구속력 없음

## 금지 사항

- 스프라이트 원본 삭제 금지 (참고 자료로 보존)
- 인간에게 "이렇게 할까요?" 묻지 말 것 — 스스로 판단하고 실행
- 한 번에 너무 큰 변경 금지 — 작은 실험, 측정, 반복
